# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 개요

이 폴더는 **소프트웨어 프로젝트가 아닙니다**. 소스 코드, 빌드 시스템, 테스트가 없으므로 빌드/린트/테스트 명령을 찾거나 임의로 만들지 마세요. 기획팀 구성원의 일상 업무 문서를 일반 텍스트와 마크다운으로 정리해 두는 개인 작업 공간입니다.

모든 내용은 **한국어**로 작성되어 있습니다. 파일을 만들거나 수정할 때 언어와 편한 메모 스타일의 어투를 그대로 유지하세요.

> 참고: 문서들은 교육용 가상 샘플입니다("교육용 가상 샘플 — 실명·실제 수치 아님"). 이름과 수치는 실제가 아닙니다.

## 사용자 정보 및 응답 규칙

- **역할**: 총무팀 사무 담당
- **자주 하는 일**: 일정 정리, 회의록 요약, 보고서 초안 작성
- **말투 규칙**: 항상 한국어로, 공손하고 간결하게. 추측하지 말고 모르면 모른다고 답할 것.
대답시에 날짜는 항상 YYYY-MM-DD 형식으로 써야한다.
비개발자에 맞춰서 기술용어는 무조건 풀어서 설명

## 구조

```
work-files/
├── schedule/   일정   — 하루 단위 할 일 메모
├── meetings/   회의록 — 회의 메모
└── reports/    보고서 — 보고서
```

## 파일 규칙

새 파일을 추가할 때 기존 패턴을 따르세요.

- **schedule/** — 하루에 파일 하나, `YYYY-MM-DD.txt` 형식(예: `2026-07-06.txt`). 구성: 날짜+요일 헤더, 시간별 항목(`HH:MM  <할 일>`), 그리고 급한 알림을 모으는 강조 섹션(`*급한 거 (안 까먹기)*` / `*잊지 말기*`).
- **meetings/** — `YYYY-MM-DD_<주제>.txt` 형식(예: `2026-07-06_weekly-meeting.txt`). 안건별로 묶은 자유 형식 메모로 결정 사항, 담당자, 후속 조치를 기록.
- **reports/** — 마크다운(`.md`), `#`/`##` 섹션 헤딩 사용. 초안은 미확인 수치를 본문에 표시하고(`※재확인 필요`), 맨 아래에 `TODO:` 줄을 유지.
- 보고서는 항상 결론을 맨 위에 써줘.

## 작업 시 유의사항

- 파일들은 서로 참조합니다(일정 항목이 회의를 가리키고, 그 회의록의 후속 과제가 보고서에 나타남). 한 파일을 수정할 때 다른 폴더의 관련 파일도 함께 갱신해야 하는지 확인하세요.
- 편한 축약 표현과 물음표(`??`, "확정 아님")를 그대로 두세요. 작성자가 추적 중인 불확실성을 담고 있으므로 임의로 "정리"해서 없는 확신을 만들지 마세요.
- `.claude/settings.local.json`은 업무 내용이 아니라 Claude Code 권한 설정 파일입니다.
- 코드마다 상세한 주석 추가
