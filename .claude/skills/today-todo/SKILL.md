# 오늘 할 일 정리 (today-todo)

`work-files/schedule`(일정)와 `work-files/meetings`(회의록)를 읽고, **오늘 기준**으로 챙겨야 할 일을 세 갈래로 나눠 정리한다.

## 절차

1. 오늘 날짜를 확인한다 (형식은 `YYYY-MM-DD`).
2. `work-files/schedule` 폴더의 파일을 읽는다.
   - 오늘 날짜 파일(`YYYY-MM-DD.txt`)을 우선 읽고, 앞으로 며칠 치 파일도 함께 훑어 다가오는 일을 파악한다.
   - `*급한 거 (안 까먹기)*` / `*잊지 말기*` 강조 섹션을 특히 챙긴다.
3. `work-files/meetings` 폴더의 최근 회의록을 읽어, 담당자·후속 조치·마감이 적힌 항목을 뽑는다.
4. 아래 세 갈래로 구분해 정리한다.

## 출력 형식

머리말로 오늘 날짜(`YYYY-MM-DD` + 요일)를 한 줄 밝힌 뒤, 다음 세 구분으로 보여준다.

- **오늘 마감** — 오늘 안에 끝내야 하는 일, 오늘 날짜의 일정, 오늘까지 마감인 후속 과제.
- **곧 다가오는 일** — 며칠 내 예정된 일정·마감(날짜를 함께 표기).
- **회의에서 나온 할 일** — 회의록의 후속 조치. 가능하면 담당자와 마감을 함께 적는다.

## 규칙 (CLAUDE.md 준수)

- 항상 한국어로, 공손하고 간결하게. 날짜는 `YYYY-MM-DD` 형식으로 쓴다.
- 추측하지 않는다. 문서에 없으면 모른다고 한다.
- `??`, "확정 아님", `※재확인 필요` 같은 미확정 표시는 확정 사실로 바꾸지 말고 "미확정"·"확인 필요"로 남긴다.
- 해당 구분에 항목이 없으면 "해당 없음"으로 표시한다.
- 각 항목 끝에 근거가 된 파일이 어디인지 필요하면 짧게 덧붙인다.
