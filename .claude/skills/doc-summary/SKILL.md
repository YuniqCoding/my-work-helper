# 문서 요약 (doc-summary)

`work-files/reports` 폴더의 보고서 파일들을 읽고, 전체 문서에 대한 핵심 요약을 **3줄**로 제공하고 **핵심 키워드**를 함께 추출한다.

## 절차

1. `work-files/reports` 폴더가 있는지 확인한다.
   - 없으면 먼저 폴더를 생성한다 (PowerShell: `New-Item -ItemType Directory -Force -Path "work-files\reports"`).
2. 폴더 안의 문서 파일(`.md`, `.txt` 등)을 모두 읽는다.
   - 파일이 하나도 없으면 "요약할 보고서가 없습니다"라고 답하고 종료한다.
3. 읽은 모든 문서를 종합해 **핵심만 3줄**로 요약하고, 이어서 **핵심 키워드**를 추출한다.

## 요약 규칙

- 항상 한국어로, 공손하고 간결하게 작성한다.
- 요약은 **정확히 3줄**을 넘기지 않는다. 세부 수치 나열보다 핵심 내용·상태 위주로 정리한다.
- 요약 뒤에 **핵심 키워드**를 짧은 구 형태로 4~6개 뽑아 함께 제시한다.
- 문서에 `※재확인 필요`, `TODO`, `??`, "확정 아님" 같은 미확정 표시가 있으면, 확정된 사실처럼 쓰지 말고 "미확정"·"확인 필요"로 남긴다.
- 문서에 없는 내용은 추측하지 않는다. 모르면 모른다고 한다.
- 요약 앞에 어떤 파일들을 근거로 삼았는지 파일명을 한 줄로 밝힌다.
